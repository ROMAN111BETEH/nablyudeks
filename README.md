# НАБЛЮДЕКС Landing + Admin (Next.js)

Продажный лендинг на Next.js с белой темой, SSR/SEO, формой заявки и админкой.

## Что реализовано

- Sticky хедер с логотипом, меню и выпадающим `Домофоны`.
- Плавный скролл к нужным секциям.
- Горизонтальные карточки товаров со свайпом/скроллом влево и сортировкой по цене.
- Избранные товары (кнопка "Понравилось") с отображением миниатюр в форме заявки.
- Форма заявки: обязательный телефон (с маской), Telegram, email, имя, комментарий.
- Простая капча, чекбокс согласия, модалка "Спасибо, заявка принята".
- Отправка заявки в Telegram-бота (включая выбранные товары, дату, UTM, контакты).
- Футер с полной контактной информацией.
- Блоки "Преимущества" и FAQ.
- Юридические тексты в модалках.
- Админка `/admin`:
  - логин/пароль (через env)
  - редактирование всего контента, SEO, контактов
  - визуальный редактор юридических модалок
  - CRUD категорий и карточек товаров
  - drag-and-drop порядок категорий/товаров
  - загрузка изображений в Supabase Storage

## Технологии

- Next.js (App Router, TypeScript)
- Supabase (Postgres + Storage)
- Telegram Bot API

## Запуск

1. Установить зависимости:
   ```bash
   npm install
   ```
2. Создать `.env.local` на основе `.env.example`.
3. В Supabase выполнить SQL из `supabase/schema.sql`.
4. Запустить dev:
   ```bash
   npm run dev
   ```
5. Открыть:
   - сайт: `http://localhost:3000`
   - админка: `http://localhost:3000/admin/login`

## Деплой на Vercel

1. Подключить репозиторий к Vercel.
2. Добавить все переменные из `.env.example` в Vercel Project Settings.
3. Выполнить `supabase/schema.sql` в проекте Supabase.
4. Убедиться, что bucket `products` создан и публичный.
